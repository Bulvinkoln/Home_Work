# Инструкция по работе с GIT

Скачать программу **"GIT"** можно, пройдя по следуйщей ссылке:

https://git-scm.com/downloads

## Инициализация репозитория

Для начала необходимо вызвать терминал. Все команды мы будем вводить в терминале.

Если в терминале скопилось большое количество команд и это делают работу в нем не удобной, можно вызвать команду
'''
clear
'''
С помощью нее программа удалит все предыдущие команды.

Следующим шагом вызываем команду
'''
git status
'''
Результатом этой комманды является строка с номером версии установленной вами программы **"GIT"**.

Для инициализации репозитория введите команду 
'''
git init
'''
Затем используя команду
'''
git status
'''
мы видим статус нашего документа. Эта комманда также используется для отслеживания изменений в файле.

## Добавление файла в коммит

После изменения файла, можно сделать сохранение или иначи *commit*, чтобы в дальнейшем можно было откатится к этому сохранению. Для этого необходимо сначала сохранить изменения в файле - необходимо нажать сочетание клавиш
'''
ctrl+s
'''
После этого добавить файл в GIT с помощью команды
'''
git add имя файла
'''
Затем необходимо создать сам *commit*. Для этого необходимо ввести команду
'''
git commit -m ""
'''
В кавычках необходимо указать имя *commit(а)*, для того, чтобы в будущем легко понять, что за изменения сохранены в данном *commit*.

## Переключение coomit(ов)

Если необходимо вернутся к предыдущему commit(у), необходимо узнать имя этого commit(а). Для этого необходимо ввести команду 
'''
git log
'''
И найти тот coomit, который вам необходим. Нам понадобятся первые 4 символа.
Затем вводим команду 
'''
git checkout № commit(а)
'''
И нажимаем *Tab*. Клаивша *Tab* полностью указывает номер за нас. Затем нажимаем *enter* и автоматически попадаеи к предыдущему commit(у).

## Сравнение commit(ов)

С помощью команды 
'''
git diff
'''
Мы можем сравнить два commit(а) между собой.

## Работа с разными ветками

**"GIT"** позволяет нам разветвлять наш файл. 

Чтобы увидеть в какой ветке мы работаем сейчас необходимо вызвать команду
'''
git branch
'''
Если мы работаем в основной ветке - мы увидим следующий результат
'''
   *master
'''

Для создания новой ветки необходимо ввести команду 
'''
git branch имя ветки
'''

Чтобы проверить создалась новая ветка или нет, повторяем комманду '''git branch'''

В результате мы увидим, что нашу основную ветку * master, она будет подсвечена зеленым, и нашу новую ветку. Звездочка перед названием ветки и подсветка зеленым цветом говорит о том, что в данный момент мы работаем в этой ветке.

Для переключения между ветками используем ту же самую комманду 
'''
git branch
'''
но после branch добавляем имя нужной нам ветки.

Если нам необходимо объединить две ветке необходимо вызвать комманду
'''
git merge имя ветки
'''
**ВАЖНО!** Всегда вы будет включать новую ветку в ветку master. Ветка *master* является основной всегда.

После объединения можно удалить ненужную вам ветку. Для этого вызываем комманду git branch -d имя ветки 
'''
git branch -d имя ветки
'''

## GitHub

Для работы с удаленныит репозиториями необходимо зарегестрироваться на портале **GitHub**. Данный портал выполняет функцию хранения ваших проектов в виде репозиториев, а также позволяет людям делиться своими репозиториями для совместной работы над ними. 

Для регестрации в браузере набираем **GitHub** и регестрируемся при помощи адреса своей электронной почты.

## Работа с удаленными репозиториями - Git pool, push, clone, pull request  

Для создания копии чужого репозитория необходимо открыть этот репозиторий и найти вкладку *fork*. Нажав ее ищем вкладку *code* и копируем ссылку на этот репозиторий. Затем в терминале ввести команда
'''
git clone
'''
и в конце вставить эту ссылку.

С помощью команды 
'''
git push
'''
мы можем отправить наш репозиторий в свой аккаунт на **GitHub**.

С помощью команды 
'''
git pull
'''
мы делаем *merge* с версией репозитория, которая находиться в **GitHub**.

**Pull request** - это чужой репозиторий, в который вы внесли свои изменения. Для его содания мы "клонируем" чужой репозиторий (как это сделать описано выше). Свои изменения вносим в новую ветку. Затем отправляем репозиторий в **GitHub** и собственно делаем **Pull request** с помощью специальной вкладки.
